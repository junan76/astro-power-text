Small projects still benefit from a consistent Git rhythm. The workflow can stay simple: make one focused change, run the relevant command, then commit with a clear subject.

Examples:

- `Add archive page`
- `Fix project article filtering`
- `Update site metadata`

The subject should describe the change, not the activity. Prefer `Add RSS feed` over `Work on RSS`.

## Review checklist

Before committing, check the rendered page, run the build, and scan the diff for unrelated edits.
